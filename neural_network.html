1. Neural network with NumPy¶

The objective is to learn the binary XOR function with a neural network. As seen in class, a neural network with only one layer cannot predict the XOR function. It has no hidden layer and no activation function, and can therefore only classify linearly separable sets. However the Universal Approximation Theorem states that a 2-layer neural network (with non-linear activation functions) can approximate any function.

# Each data point in the XOR dataset has two variables (or features), x1 and x2, that can be either 0 or 1
# The label to predict is their XOR combination (1 if x1 and x2 are different, 0 otherwise)
n_samples = 500
X_train = np.random.randint(low=0, high=2, size=(n_samples, 2)).astype('float32') 
d_train = np.logical_xor(X_train[:, 0], X_train[:, 1]).astype('int')

# To make it slightly more challenging, we add a bit of gaussian noise to the data points.
std = 0.1
X_train += std * np.random.randn(n_samples, 2)
X_train = X_train.T

Activation functions¶

The feed-forward algorithm works as follows:

  1. Each neuron $i$ in the hidden layer $l$ receives the weighted sum of the input values: $$a_i^l=\sum_j h_j^{l-1} w_{ij}^l + b^l_i$$
  2. Each neuron calculates an output by using an activation function $g_1$ (typically sigmoid or ReLU): $$h_i^l= g_1(a_i^l)$$
  3. Each neuron $i$ in the output layer $L$ receives the weighted sum of the last hidden layer: $$a_i^L =\sum_j h_j^{L-1} w_{ij}^{L} + b^L_i$$
  4. The output layer neuron calculates an output by using possibly another activation function $g_2$ (depending on the application): $$y_i= g_2(a_i^L)$$

In case of classification, logistic activation functions such as sigmoid or softmax are usually used for the output layer.

Loss function¶

To learn the parameters of the network we must compute the total loss of the forward pass ${\cal L}$.

It is computed by averaging the individual loss $L(y(\mathbf{x}), d)$ for each training sample $(\mathbf{x}, d)$ : $${\cal L} = \frac{1}{N} \sum_{(\mathbf{x},d)\in T} L(y(\mathbf{x}), d).$$ where $y(\mathbf{x})$ is the output of the network (prediction) and $d$ is the ground truth label.

Different loss functions $L$ can be used, depending on the problem. Here, because we are working on a binary classification problem, we will use the binary cross-entropy loss: $$L(y, d)= - (d\log (y)+ (1-d)\log(1-y))$$ and its derivative with respect to the outputs, which will be used for backpropagation: $$\frac{\partial L(y, d)}{\partial y} = -\frac{d}{y} + \frac{1 - d}{1 - y}$$

def crossentropy(y, d):
    # Return a scalar corresponding to the total loss.
    # It should be the average of the binary cross-entropy
    # for each element in `y` (predictions) and `d` (ground truth labels)
    return np.sum(-(d*np.log(y)+(1-d)*np.log(1-y)))/y.shape[0]
    

def d_crossentropy(y, d):
    # Return a vector of partial derivatives of the binary cross-entropy loss
    # with respect to each of the predictions.
    return -d/y + (1-d)/(1-y)

3. Automatic differentiation - Backpropagation¶

A central feature in PyTorch is the autograd package. It provides tools performing automatic differentiation for all operations based on Torch tensors. This means that the gradients of such operations will not require to be explicitly programmed. It uses a define-by-run framework that computes these gradients dynamically during runtime. This is really useful to apply the backpropagation algorithm, since autograd will deduce automatically from the succession of forward operations what are the corresponding backward updates.

A Torch tensor stores a flag requires_grad. By default, this attribute is set to False. If you set it to True, autograd will track all operations where this Tensor is involved. Any other tensor resulting from operations involving this tensor will have its requires_grad attribute set to True automatically. When you finish your computation you can call .backward() and have the gradients computed automatically for all these tensors. The gradient for these tensors will be accumulated into the .grad attribute.

        self.labels = (self.data.sum(dim=1) == 1).to(torch.int)
        self.data += self.std * torch.randn(self.data.shape)

    def __len__(self):
        # Number of data points we have. Alternatively self.data.shape[0], or self.labels.shape[0]
        return self.n_samples

    def __getitem__(self, index):
        # Return the index-th data point of the dataset
        # If we have multiple things to return (data point and label), we can return them as tuple
        data_point = self.data[index]
        data_label = self.labels[index]
        return data_point, data_label

class XORClassifier(nn.Module):

    def __init__(self, n_inputs, n_hidden, n_outputs):
        super().__init__()
        # Initialize the modules we need to build the network
        self.linear1 = nn.Linear(n_inputs, n_hidden)
        self.act_fn1 = nn.Sigmoid()
        self.linear2 = nn.Linear(n_hidden, n_outputs)
        self.act_fn2 = nn.Sigmoid()

    def forward(self, x):
        # Perform the calculation of the model to determine the prediction
        h1 = self.linear1(x)
        a1 = self.act_fn1(h1)
        h2 = self.linear2(a1)
        y = self.act_fn2(h2)
        return y

Let us now implement the training loop that performs the forward pass, calculates the loss, and apply backpropagation to update the weights.

def train_model(model, optimizer, dataloader, loss_fun, num_epochs=200):
    # Set model to train mode
    model.train()

    # Training loop
    for epoch in range(num_epochs):
        for (data_inputs, data_labels) in dataloader:

            ## Step 1: Run the model on the input data
            preds = model(data_inputs)
            preds = preds.squeeze(dim=1) # Output is [Batch size, 1], but we want [Batch size]

            ## Step 2: Calculate the loss
            loss = loss_fun(preds, data_labels.float())

            ## Step 3: Perform backpropagation
            # Before calculating the gradients, we need to ensure that they are all zero.
            # Otherwise, the gradients would be accumulating to the previous ones.
            optimizer.zero_grad()
            # Perform backpropagation
            loss.backward()

            ## Step 4: Update the parameters
            optimizer.step()
